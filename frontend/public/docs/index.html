PromptSense Documentation

PromptSense is a multi-tenant LLM guardrail platform that sits between your application and your AI provider — screening every prompt and response for safety, compliance, and quality issues in real time.

⚡ Quickstart 🔗 Proxy Endpoint 🛡 Guardrails 📡 API Reference

🛡 10 built-in guardrails

PII, prompt injection, toxicity, secrets detection, hallucination checks — active the moment you connect.

⚡ 6 LLM providers

Anthropic, OpenAI, Azure, Gemini, Mistral, Cohere — or bring your own via custom endpoint.

📊 Real-time analytics

Pass rates, flag distribution, provider latency, and cost tracking — all in one live dashboard.

🔔 Webhook alerts

Fire to Slack, PagerDuty, or any HTTP endpoint the instant a prompt is blocked or flagged.

👥 Team RBAC

User, Developer, and Administrator roles with granular permission gating across your org.

📜 Immutable audit log

Every prompt, response, and guardrail decision captured — exportable to CSV for compliance review.

Quickstart

Get PromptSense protecting your LLM traffic in under 5 minutes.

1

Create an account

Sign up at promptsense.io/register. Registration automatically creates your organization and seeds it with 10 system guardrails and a default policy.

2

Connect an LLM provider

Go to Integrations in the dashboard. Click Connect next to your provider (e.g. Anthropic), paste your API key, select a model, and click Save connection. Keys are encrypted at rest with AES-256-GCM.

3

Create an API key

Go to API Keys and click Create key. Copy the key — it won't be shown again. This key is used to authenticate SDK and direct API calls.

4

Point your app at the proxy

Replace your direct LLM API call with a single POST to the PromptSense proxy:

curl -X POST https://your-backend.onrender.com/api/orgs/{orgId}/proxy \
  -H "X-PromptSense-Key: ps_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Summarise this contract for me.", "provider": "anthropic"}'

PromptSense evaluates the prompt against your active guardrails, routes it to Anthropic (or your chosen provider), screens the response, and returns the result — all in a single call.

How It Works

Every request flows through a 5-stage pipeline:

StageWhat happens
1. AuthenticateJWT bearer token or X-PromptSense-Key API key is verified.
2. Rate limitPer-org RPM cap enforced based on your plan (Redis-backed in production).
3. Input guardrailsThe prompt is tested against all enabled guardrails with type: input or type: both. Blocked prompts never reach the LLM.
4. LLM callPromptSense forwards the prompt to your configured provider using your stored (encrypted) API key.
5. Output guardrailsThe LLM response is tested against guardrails with type: output or type: both. Flagged responses can be blocked, warned, or logged.
6. Audit + webhooksEvery event — pass or block — is written to the immutable audit log and any matching webhooks are fired asynchronously.

Guardrails

Guardrails are regex-based or semantic rules that screen prompts and responses. Each guardrail has a type (input / output / both), a severity, and an action.

System Guardrails (pre-seeded)

NameTypeWhat it catchesDefault action
PII detectionBothEmails, SSNs, credit card numbers, phone numbers in prompts and responses.Block
Prompt injectionInputInstructions attempting to override the system prompt (e.g. "ignore previous instructions").Block
Toxicity filterBothProfanity, hate speech, and harmful language patterns.Block
Secrets detectionBothAPI keys, tokens, and credentials appearing in traffic.Block
Hallucination checkOutputPatterns typical of model confabulation (e.g. fabricated citations).Warn
Output length capOutputResponses exceeding a configured character limit.Log

Guardrail Fields

FieldValuesDescription
typeinput output bothWhich side of the LLM call to evaluate.
severitycritical high medium lowSurfaced in analytics and audit log for prioritisation.
actionblock warn logblock — returns 422 and stops the request. warn — returns the response with a warnings field. log — passes silently but writes the flag to the audit log.
patternRegex stringCase-insensitive JavaScript regex tested against the text. Use the built-in pattern tester in the dashboard before saving.

Custom Guardrails

Go to Guardrails → New guardrail. Write a regex pattern for your domain — for example:

# Block competitor brand names
(acme|rival-corp|competitor-name)

# Catch attempts to extract training data
(repeat after me|tell me your (system|instructions|prompt))

# Detect internal code names
\b(project-?nova|operation-?bluesky)\b
Tip: Use the Test pattern field in the guardrail editor to validate your regex against sample inputs before enabling it in production.

Policies

A policy is a named collection of guardrails that can be applied as a bundle. Instead of managing individual rules per integration, create a policy (e.g. "Healthcare compliance") and attach the guardrails that matter for that context.

  • Each org can have multiple policies.
  • Policies can be marked active — the proxy always evaluates active policies.
  • A default policy named "Default policy" is seeded at registration containing the 6 core system guardrails.
  • Requires Developer role or higher to create or modify.

Endpoint URL for Anthropic

When configuring Anthropic, leave the Endpoint URL as the default: https://api.anthropic.com/v1/messages. Do not append a path — PromptSense constructs the full request internally.

Proxy Endpoint

The proxy is the single entry point for all LLM traffic. Send your prompt here and PromptSense handles guardrail evaluation, provider routing, and response screening.

POST /api/orgs/{orgId}/proxy Run a prompt through guardrails and your LLM provider

Authentication

The proxy accepts two authentication methods:

  • API Key (recommended for server-to-server): X-PromptSense-Key: ps_your_key
  • JWT Bearer token (for dashboard / user sessions): Authorization: Bearer <token>

Request body

{
  "prompt": "Summarise this contract for me.",
  "provider": "anthropic",         // optional — uses your default connected provider
  "model": "claude-sonnet-4-20250514", // optional — overrides provider default model
  "systemPrompt": "You are a legal assistant.", // optional
  "maxTokens": 1024              // optional
}

Success response (200)

{
  "output": "Here is a summary of the contract...",
  "passed": true,
  "provider": "anthropic",
  "model": "claude-sonnet-4-20250514",
  "latencyMs": 834,
  "tokensUsed": 312,
  "inputFlags": [],
  "outputFlags": []
}

Blocked response (422)

{
  "error": "Prompt blocked by guardrail: PII detection",
  "guardrail": "PII detection",
  "severity": "critical",
  "passed": false
}

Full example — Node.js

const response = await fetch(
  `https://your-backend.onrender.com/api/orgs/${orgId}/proxy`,
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-PromptSense-Key': process.env.PROMPTSENSE_API_KEY,
    },
    body: JSON.stringify({
      prompt: userMessage,
      provider: 'anthropic',
    }),
  }
);

const data = await response.json();

if (!response.ok) {
  // Prompt was blocked — handle gracefully
  console.error('Blocked:', data.guardrail);
} else {
  console.log('Response:', data.output);
}

Full example — Python

import requests, os

resp = requests.post(
    f"https://your-backend.onrender.com/api/orgs/{org_id}/proxy",
    headers={
        "X-PromptSense-Key": os.environ["PROMPTSENSE_API_KEY"],
    },
    json={"prompt": user_message, "provider": "anthropic"},
)

if resp.status_code == 422:
    print("Blocked:", resp.json()["guardrail"])
else:
    print("Response:", resp.json()["output"])

Connecting to a chatbot (input + output guardrails)

To add PromptSense guardrails to an existing chatbot, replace your direct LLM call with the proxy endpoint:

// Before — direct Anthropic call
const result = await anthropic.messages.create({
  model: 'claude-sonnet-4-20250514',
  messages: [{ role: 'user', content: userInput }],
});

// After — PromptSense proxy (guardrails applied automatically)
const result = await fetch(`/api/orgs/${ORG_ID}/proxy`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-PromptSense-Key': PROMPTSENSE_KEY,
  },
  body: JSON.stringify({ prompt: userInput, provider: 'anthropic' }),
}).then(r => r.json());

// result.output contains the LLM response
// result.passed is false if guardrails blocked the response

Prompt Templates

Templates let you define reusable system prompts with variable substitution. Create a template once and reference it by name across your integrations.

  • Templates use {{variable}} syntax for dynamic values.
  • Stored per-org, versioned, and auditable.
  • Requires Developer role to create or modify.
# Example template: "Legal assistant"
You are a legal assistant for {{company_name}}.
Your jurisdiction is {{jurisdiction}}.
Only discuss topics related to {{practice_area}}.
Never provide specific legal advice — always recommend consulting a licensed attorney.

Audit Log

Every request through the proxy is written to an immutable audit log — whether it passed or was blocked. This provides a tamper-evident record for compliance and incident investigation.

Audit record fields

FieldDescription
created_atTimestamp of the event.
providerLLM provider used (anthropic, openai, etc.).
modelModel name.
passedWhether all guardrails passed.
latency_msTotal round-trip time in milliseconds.
tokens_usedToken count for the request.
input_flagsArray of guardrail names that flagged the input.
output_flagsArray of guardrail names that flagged the output.
input_textThe original prompt.
output_textThe LLM response (empty if blocked at input).

CSV Export

Export up to 10,000 records as CSV for compliance review, SIEM ingestion, or offline analysis. Available to Pro and Enterprise plans.

GET /api/orgs/{orgId}/audit/export

Webhooks

Webhooks fire an HTTP POST to your chosen endpoint whenever a guardrail event occurs. Use them to push alerts to Slack, PagerDuty, or your own SIEM.

Event triggers

  • prompt.blocked — a prompt was blocked by a guardrail.
  • response.blocked — a response was blocked after the LLM call.
  • guardrail.flagged — a warn-action guardrail fired (but did not block).

Webhook payload

{
  "event": "prompt.blocked",
  "orgId": "3fa85f64-...",
  "guardrail": "PII detection",
  "severity": "critical",
  "provider": "anthropic",
  "timestamp": "2026-04-11T14:32:00Z"
}
SSRF protection: Webhook URLs are validated against a blocklist of private IP ranges before being saved. Only public HTTPS endpoints are accepted.

Gauntlet

Gauntlet is PromptSense's automated adversarial test suite. It fires a curated library of attack prompts at your guardrail configuration and reports which ones were blocked vs. which slipped through.

Test categories

  • Prompt injection — jailbreaks, role-play overrides, instruction extraction attempts.
  • PII exfiltration — prompts that try to leak personal data.
  • Toxicity bypass — encoded or obfuscated harmful content.
  • Secrets leakage — attempts to surface API keys or credentials.
  • Adversarial inputs — edge cases designed to confuse regex-based filters.

Running a Gauntlet test

  1. Go to Gauntlet in the dashboard.
  2. Select the test categories you want to run.
  3. Click Run gauntlet.
  4. Review the results table — each test shows pass/fail, the guardrail that caught it (or "Missed"), and the exact test prompt.
Tip: Run Gauntlet after adding or modifying guardrails to verify your changes catch the intended attack patterns without over-blocking legitimate traffic.

Playground

The Playground lets you test prompts manually against your live guardrail configuration — without touching production traffic or your request quota.

  • Select any connected provider and model.
  • Type or paste a prompt and click Run.
  • See the full guardrail evaluation result: which rules fired, the LLM response, latency, and token count.
  • Playground calls are written to the audit log for traceability.

Roles & Permissions

RoleWhat they can do
User View analytics, audit log, guardrails, and policies. Use the Playground. Cannot modify configuration.
Developer Everything a User can do, plus: create/edit guardrails, policies, templates, webhooks, API keys, and configure LLM providers.
Administrator Full access. Invite and remove team members, change roles, manage billing and org settings.

Role changes take effect immediately. Superusers (platform staff) bypass all role checks.

API Keys

API keys are used to authenticate server-to-server calls to the proxy endpoint. They do not expire by default but can be revoked instantly.

  • Keys are displayed only once at creation — copy and store them securely.
  • Keys are stored as SHA-256 hashes — PromptSense cannot recover a lost key.
  • Requires Developer role or higher to create.
  • Pass keys in the X-PromptSense-Key header.
Never expose API keys client-side. Always call the proxy from your backend server, not from browser JavaScript.

Team Members

Invite team members by email from Members in the dashboard. Invitees receive an email with a registration link pre-associated to your org.

  • Assign a role (User, Developer, or Administrator) at invite time.
  • Change roles or remove members at any time — requires Administrator.
  • Member limits apply based on your plan (3 on Starter, 15 on Pro, unlimited on Enterprise).
